Semantic HTML and Screen Readers

The word "semantic" means "meaning". The HTML elements have names and some of these name imply functionality and some don't. For example, h1 means Header 1. We know there is only one h1 element on a web page. If a screen reader encounter this header, it can deduce that This is the primary header for the web page and create a meaningful response based on that.

On the other hand, the div element was created to help with page layout. It doesn't provide any meaninful informations about the content of the document. Ascreen reader will ignore a div tag.

In this module, we're going to look at an aspect of Semantic coding that affects interactivity. An HTML button looks like a button by default. It can be styled to looked more interactive. Semantically, it is the way to code a clickable element in HTML. In this example, we'll see that we can make any item clickable in the sense that we can trigger an action based on a click. The problem is that a screen reader won't find a a clickable element unless it's a button or a link. We'll see how to check for this name in the Chrome browser.
