Semantic HTML and Screen Readers

The word "semantic" means "meaning". The HTML elements have names and some of these name imply functionality and some don't. For example, h1 means Header 1. We know there is only one h1 element on a web page. If a screen reader encounter this header, it can deduce that this is the primary header for the web page and create a meaningful response based on that.

On the other hand, the div element was created to help with page layout. It doesn't provide any meaninful informations about the content of the document. A screen reader will ignore a div tag.

In this module, we're going to look at an aspect of Semantic coding that affects interactivity. An HTML button looks like a button by default. It can be styled to looked more interactive. Semantically, it is the way to code a clickable element in HTML. In this example, we'll see that we can make any item clickable in the sense that we can trigger an action based on a click. The problem is that a screen reader won't find a clickable element unless it's a button or a link.

Interact with the Buttons in the Code Sandbox

In the code sandbox below, we show that we can style a div to look like a button. It's also possible to add a click event to any element using JavaScript or an onclick HTML attribute. This will cause problems for a person using a screen reader though.

Open the sandbox in a new window and ry clicking on the buttons in the sandbox. You'll see that they all respond to click events with a JavaScript alert.

ARIA is a technique for adding semantics to unsemantic web tags like div. Just add attribute role="button" to the div tags and these elements will be understood to semantically behave as buttons.
